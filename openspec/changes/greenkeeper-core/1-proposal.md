# 1-proposal.md (Greenkeeper Core)

## The Problem
Lasse needs a stealthy, multi-agent automation setup on his corporate Mac to process Teams messages, Planner tasks, OpenCode tasks, and emails without triggering EDR (CrowdStrike/Defender) or enterprise Data Loss Prevention (DLP) flags. Direct Graph API polling using local tokens is blocked by Conditional Access policies. Furthermore, the system must support two-way interaction (confirming tasks and triggering local scripts) without opening inbound ports or exposing a shell to LLM-generated strings.

## The Solution
A "Blood-Brain Barrier" architecture utilizing a Power Automate Hybrid model:
1. **Cloud Egress:** Benignly named Power Automate flows batch M365 data into JSON files and save them to a specific OneDrive folder.
2. **Local Ingestion:** A low-footprint Node.js/TypeScript daemon (`orchestrator.ts`) uses `fs.watch` to detect new files arriving via standard OneDrive OS sync.
3. **The Sanitizer:** A 3-stage pipeline (Regex -> Azure AI Foundry -> Regex) strips all corporate IP, names, and credentials from the data before summarization.
4. **Secure Delivery:** The sanitized payload is encrypted locally with AES-256-GCM and pushed to a dumb Cloudflare Worker relay. The relay utilizes "consume-on-read" (DELETE immediately after GET) to prevent replay attacks and minimize exposure windows.
5. **The Glass (Dashboard):** A Progressive Web App (PWA) running locally on the personal device that fetches the encrypted blob, decrypts it in-browser using the Web Crypto API, and renders the data securely.
6. **Inbound Intent Channel:** The PWA sends integer-only encrypted intent codes back through the Cloudflare Worker. The local daemon polls this worker, mapping integers (via `aliases.json`) to hardcoded, parameter-less local execution scripts, entirely eliminating Arbitrary Code Execution (ACE) vectors.

## Key Security Properties
- **Zero Inbound Ports:** The system relies entirely on outbound polling to Cloudflare and passive OneDrive OS sync.
- **Strict Data Sanitization:** The 3-stage blood-brain barrier ensures no raw corporate IP leaves the endpoint.
- **Consume-on-Read:** Encrypted payloads are burned upon retrieval, closing the replay window.
- **Zero-Trust Browser Rendering:** The PWA strictly forbids `innerHTML` or dynamic script execution to prevent XSS attacks originating from malformed or malicious Teams messages.
- **Immutable Execution (No ACE):** The `OpenCode Runner` only accepts pre-mapped integer intents (`aliases.json`) and never executes strings dynamically generated by an LLM or an external source.
- **Volume Threshold Monitoring (Panic Switch):** A permanent halt mechanism (`panic.lock`) prevents rogue flows from mass-exfiltrating data.
